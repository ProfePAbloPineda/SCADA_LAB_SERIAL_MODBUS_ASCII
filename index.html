🔧 Guía de Laboratorio SCADA

PLC Delta SE + KEPServerEX + InfileLink

🎯 Objetivos del Laboratorio

  • Configurar la comunicación entre un PLC Delta DVP-SE y una estación PC mediante puerto virtual USB y protocolo Modbus Serial ASCII
  • Enlazar KEPServerEX con el PLC y mapear variables de proceso para exposición vía OPC/Modbus
  • Desarrollar una interfaz SCADA minimalista y funcional en InfileLink para monitoreo y control básico
  • Configurar el COM2 del PLC para lectura de temperatura mediante módulo DVP04TC conectado a red RS485
  • Aplicar en el programa del PLC lógica con salidas retenidas, temporizadores, contadores y una rutina aislada de conversión de dato entero a flotante

📦 Equipos y Software Requerido

PLC Delta DVP-SE (con firmware compatible Modbus)
Cable USB a PLC (genera puerto COM virtual en Windows)
Módulo DVP04TC (lectura de temperatura, interfaz RS485)
Sensor Transmisor de temperatura compatible RS485
Software PLC WPLSoft / ISPSoft
Gateway SCADA KEPServerEX (driver Modbus Serial)
HMI/SCADA InfileLink
PC Windows 10/11, drivers USB-Serial

🎛️ Planteamiento del Problema

Contexto Industrial

Sistema de dosificación y control térmico para un tanque de mezcla en una planta de procesos.

Requisitos Funcionales del Programa PLC

1. Salidas retenidas (latch/unlatch):
Bomba de llenado y válvula de descarga deben activarse/desactivarse mediante pulsadores START/STOP y mantener su estado hasta nueva orden o falla.
2. Temporizador (TON/TOF):
Controlar el tiempo de activación de la bomba de llenado. El temporizador debe reiniciarse automáticamente al completarse el ciclo o al presionar RESET.
3. Contador ascendente:
Contar la cantidad de ciclos de dosificación completados. Debe disponer de un conteo parcial que refleje el avance del ciclo en curso.
4. Conversión entero → flotante (rutina aislada):
Implementar una subrutina o bloque independiente que tome un valor entero y lo convierta a formato flotante IEEE-754.
⚠️ Nota importante: El estudiante deberá programar la solución en el PLC. No se proporciona código intencionalmente para fomentar el análisis funcional.
FASE 1

Configuración USB y KEPServerEX

1.1 Configuración del Puerto Virtual

  • Conectar el PLC a la PC mediante cable USB
  • Verificar en Administrador de dispositivos que se asigne un puerto COM virtual (ej. COM3)

1.2 Configuración COM1 en el PLC

  • Modo: ASCII
  • Baudrate: 9600
  • Data bits: 8, Parity: Even, Stop: 1
  • Dirección Modbus: 1

1.3 Configuración en KEPServerEX

  • Crear nuevo canal → Driver: Modbus Serial → Protocolo: ASCII
  • Seleccionar el puerto COM virtual asignado
  • Configurar parámetros idénticos al PLC
  • Crear dispositivo con Unit ID = 1
✅ Verificar comunicación con OPC Quick Client o KEPServerEX Test Client
FASE 2

Creación de Tags en KEPServerEX

Crear las siguientes variables bajo el dispositivo Modbus. Ajustar direcciones según la tabla de mapeo del PLC Delta:

Tag Tipo Dirección Descripción
PLC.START Boolean 00001 Pulsador marcha
PLC.STOP Boolean 00002 Pulsador parada
PLC.PUMP Boolean 00101 Salida retenida bomba
PLC.TIMER_STATUS Boolean T0.DN Estado del temporizador
PLC.TIMER_CURRENT Integer T0.ACC Valor actual temporizador
PLC.COUNTER_STATUS Boolean C0.DN Estado del contador
PLC.COUNTER_PARTIAL Integer C0.ACC Conteo parcial
PLC.FLOAT_VALUE Float D10-D11 Registro 32-bit flotante
FASE 3

Diseño de Pantalla SCADA en InfileLink

Requisitos de la Interfaz

  • Crear nuevo proyecto → Conectar a KEPServerEX vía OPC (DA o UA)
  • Diseñar una pantalla minimalista y funcional con:
  • Botones START, STOP, RESET (enlazar a tags booleanos)
  • Indicadores de estado para PUMP, VALVE, TIMER_STATUS, COUNTER_STATUS
  • Displays numéricos para TIMER_CURRENT, COUNTER_PARTIAL, COUNTER_TOTAL
  • Display para FLOAT_VALUE (formato %.2f)
  • Etiquetas claras, sin elementos decorativos innecesarios

Configuración de Actualización

  • Polling: 500ms
  • Deadband: 0 (para depuración)

FASE 4

Configuración COM2 - Módulo DVP04TC

Conexión Física RS485

  • Conectar el módulo DVP04TC al bus RS485 del COM2 del PLC
  • Verificar polaridad D+ / D-
  • Terminación de 120Ω si es red extendida

Configuración del COM2 en el PLC

  • Protocolo: Modbus RTU o ASCII (según especificación del DVP04TC)
  • Baudrate: 9600/19200
  • Paridad: Even/None, Bits: 8, Stop: 1
  • Dirección del transmisor: 2
  • Asignar registros internos (ej. D20-D23 por canal)

Configuración en KEPServerEX

  • Agregar nuevo dispositivo bajo el mismo canal o canal separado
  • Unit ID = 2
  • Mapear registros de temperatura (40001-40004 como Float32 o Int16)
FASE 5

Integración de Temperatura y Programa PLC

Tags de Temperatura en KEPServerEX

  • TEMP.CH1, TEMP.CH2, TEMP.CH3, TEMP.CH4 (tipo Float)
  • TEMP.MODULE_STATUS (Boolean o Word de diagnóstico)

Actualización de InfileLink

  • Agregar displays numéricos para cada canal de temperatura
  • Incluir indicador de estado del módulo
  • Mantener diseño minimalista: agrupar en panel TEMPERATURA
  • Configurar escalado si el transmisor entrega entero (ej. 0.1°C por bit)

📝 Programa del PLC

Para el programa:
El estudiante debe desarrollar el programa en ISPSoft incluyendo:
  • Lógica de salidas retenidas
  • Temporizadores TON/TOF
  • Contadores ascendentes
  • Rutina aislada de conversión entero→flotante
